# RAPO Git Workflow

## Development Flow

Feature

↓

Compile

↓

Run

↓

Review

↓

Commit

↓

Push

## Branch Naming

Use descriptive branch names such as:

- feature/robots-scanner
- feature/sitemap-scanner
- fix/output-rendering
- chore/documentation-update

## Commit Message Conventions

Use clear, concise commit messages that describe the change.

Examples:

- Add robots.txt scanner support
- Integrate sitemap reporting into output
- Improve Rich UI consistency

## Milestones

Milestones should be treated as meaningful checkpoints for feature delivery, integration, and validation.

## Release Workflow

Releases should be prepared only after verification, review, and a stable test run.

## Versioning

Versioning should remain simple and deliberate. Increment versions when meaningful changes are introduced and validated.
