# RAPO UI Style Guide

## Rich Terminal Conventions

RAPO should present information through a consistent and polished Rich terminal experience.

## Banner

Every report should begin with a clear banner that identifies RAPO and its version.

## Colors

Use a restrained and professional color palette.

- Cyan for headings and structural emphasis
- White or bright white for primary text
- Green for success states
- Yellow for warnings
- Red for important failures or high-risk notices

## Icons

Icons should be used sparingly and consistently to support readability.

Examples:

- 🎯 Target
- 🌐 DNS
- 🌍 HTTP
- 🔓 Open Ports
- 📄 WHOIS
- 🛠 Technologies
- 🔒 SSL
- 🛡 Security Headers
- 🤖 Robots.txt
- 📄 Sitemap

## Tables

Use tables for structured values where appropriate. Keep columns aligned and readable.

## Panels

Use panels to group related content into readable sections. Keep panel titles clear and concise.

## Scan Summary

The report should end with a concise summary section that communicates the scan status clearly.

## Progress Bars

If progress indicators are introduced, they should be simple, minimal, and non-distracting.

## Success Messages

Success messages should be positive and concise.

## Error Messages

Error messages should be explicit, calm, and actionable.

## Terminal Layout

The layout should remain consistent across modules. Reports should be visually organized and easy to follow.
